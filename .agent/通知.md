# 通知模块对接流程

## 1. 对接前置

通知模块分为后台管理端和 C 端会员端。后台负责站内信展示、推送记录查看、通知场景配置和超级管理员配置概览；C 端负责会员通知列表、详情、已读和未读数量。

常规 HTTP 响应统一使用 `{code, message, data}` 包装，业务数据均从 `body.data` 读取。所有 64 位 ID 建议前端按字符串处理，避免 JS 精度问题。

## 2. 核心概念

### 2.1 通知渠道

| 渠道           | 说明               | 面向端   | 是否支持已读 |
| -------------- | ------------------ | -------- | ------------ |
| `inbox`        | 后台站内信         | 后台     | 是           |
| `mini_program` | 微信小程序订阅消息 | C 端会员 | 否           |

已读是当前用户在支持已读渠道上的阅读状态，不是通知事件的全局状态。小程序订阅消息只记录投递状态，不参与未读数量，也不会因为站内信已读而同步已读。

### 2.2 通知场景与配置

系统预注册通知场景，场景只描述基础信息和支持范围；商户配置时只保存场景开关和 `channels` 列表。`channels` 内每一项就是一个渠道配置，包含端、渠道、模板和后台接收人。

当前内置场景：`reservation_created`、`reservation_cancelled`、`reservation_expired`、`reservation_arrived`、`reservation_completed`、`deposit_created`、`deposit_approved`、`deposit_rejected`、`deposit_cancelled`。

### 2.3 关联展示数据

接口返回关联 ID 时会同时返回展示对象，例如 `merchantId/merchant`、`receiverMemberId/receiverMember`、`receiverRoleIds/receiverRoles`、`receiverAccountIds/receiverAccounts`。后台站内信是当前登录账号自己的消息，不返回当前账号的 `receiverAccount`。

前端和投递记录都不会出现 `receiverOpenid`；小程序发送时后端通过会员信息实时查询 openid。

## 3. 后台站内信

### 3.0 实时提醒 Socket.IO

后台站内信支持 Socket.IO 实时提醒。连接入口：

```ts
import { io } from 'socket.io-client'

const socket = io('/adminNotify', {
  path: '/socket.io/',
  transports: ['websocket'],
  query: { token },
})

socket.on('notification.created', (payload) => {
  // payload.type = 'admin.notification.created'
  // 推荐收到后刷新未读数量，当前在消息中心时再刷新列表
})
```

当前 Go 服务端使用 `query.token` 或 `Authorization: Bearer <token>` 认证后台账号，不需要也不要传 `auth.token`。实时事件只做新站内信提醒，历史消息和断线补偿仍以 HTTP 列表接口为准。

### 3.1 查询列表

```http
GET /api/notification/v1/notifications?pageRequest.page=1&pageRequest.pageSize=20
```

后台站内信固定只返回当前账号的 `inbox` 投递结果，唯一投递字段已聚合到顶层，不返回 `deliveries`。

### 3.2 详情、已读、未读数

```http
GET /api/notification/v1/notifications/{id}
POST /api/notification/v1/notifications/read
GET /api/notification/v1/notifications/unread/count
```

标记已读请求体：

```json
{"id": 1}
```

```http
GET /api/notification/v1/preferences
```

列表会返回所有系统通知场景；商户未保存过的场景也会返回默认配置视图。

```http

```

### 5.3 更新通知配置

```http
POST /api/notification/v1/preferences_put
```

```json
{
  "sceneCode": "reservation_created",
  "enabled": true,
  "channels": [
    {
      "side": "admin",
      "channel": "inbox",
      "enabled": true,
      "providerTemplateId": "",
      "contentTemplate": "会员{member_name}已预约{store_name}，到店时间：{reservation_time}",
      "variableSchemaJson": "[\"member_name\",\"store_name\",\"reservation_time\"]",
      "miniProgramPagePath": "",
      "receiverRoleIds": [1],
      "receiverAccountIds": []
    },
    {
      "side": "app",
      "channel": "mini_program",
      "enabled": true,
      "providerTemplateId": "wx-template-id",
      "contentTemplate": "您的预约已创建：{reservation_time}",
      "variableSchemaJson": "[\"reservation_time\"]",
      "miniProgramPagePath": "pages/reservation/detail",
      "receiverRoleIds": [],
      "receiverAccountIds": []
    }
  ]
}
```

字段说明：

| 字段                            | 类型    | 说明                                    |
| ------------------------------- | ------- | --------------------------------------- |
| `sceneCode`                     | string  | 通知场景编码                            |
| `enabled`                       | bool    | 是否启用该场景                          |
| `channels[].side`               | string  | `admin` 或 `app`                        |
| `channels[].channel`            | string  | `inbox` 或 `mini_program`               |
| `channels[].enabled`            | bool    | 是否启用该渠道                          |
| `channels[].contentTemplate`    | string  | 渠道内容模板，支持 `{变量名}`           |
| `channels[].receiverRoleIds`    | int64[] | 后台渠道接收角色，仅 `admin/inbox` 使用 |
| `channels[].receiverAccountIds` | int64[] | 后台渠道接收账号，仅 `admin/inbox` 使用 |

后台模板不再单独维护；模板内容随 `channels[]` 一起保存。`mini_program` 的接收人来自业务事件携带的会员 ID，不从这里的角色或账号配置推导。

## 6. 超级管理员配置概览

```http
GET /api/notification/v1/sceneConfigOverview
```

仅超级管理员可访问。概览接口只返回所有通知场景和已配置商户数量，不返回商户配置明细，避免商户数量较多时响应过大。

```json
{
  "code": 0,
  "message": "成功",
  "data": {
    "items": [
      {
        "sceneCode": "reservation_created",
        "sceneName": "预约创建通知",
        "description": "预约创建后通知后台人员或会员",
        "configuredMerchantCount": 12
      }
    ]
  }
}
```

如需查看某个场景下具体哪些商户配置了该场景，调用分页详情接口：

```http
GET /api/notification/v1/sceneConfigMerchants?sceneCode=reservation_created&pageRequest.page=1&pageRequest.pageSize=20
```

```json
{
```

## 8. 推荐页面与注意事项

1. 后台顶部消息角标调用未读数量接口；消息中心列表调用站内信列表。
2. 推送记录页用于排查失败通知，按“推送消息 -> 渠道 -> 接收人投递明细”展示。
3. 通知配置页按场景展示基础信息和 `channels[]`，在 channel 内编辑模板和接收范围。
4. 模板变量来自业务事件 payload，模板里不存在的变量会保留原占位文本。
5. 小程序订阅消息依赖用户授权，未授权或微信返回失败时，投递记录会进入失败状态并按队列重试策略处理。
6. 前端调试接口时可以配合 Bruno 集合：`bruno/admin/10-通知模块/`、`bruno/app/13-通知模块/`。
